# RBAC Design Guide

この資料は、実務クラスタで `誰が何をできるか` を設計するときの最小ガイドです。

## まず考えること

- 人の権限と workload の権限を分ける
- 読み取り専用で十分かを先に考える
- namespace をまたぐ権限は本当に必要か確認する

## 最小権限の考え方

- まず get / list / watch だけで足りるか考える
- update / patch / delete は必要性を明示する
- CI、Argo CD、Job、運用者は別主体にする

## 実務での観点

- cluster-admin は常用しない
- 誰が何をできるかをレビュー可能にする
- 一時的な強い権限は期限つきにする
